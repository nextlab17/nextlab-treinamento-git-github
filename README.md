# Treinamento Git & GitHub - Laboratório

## README.md

```markdown
# Treinamento Git & GitHub - Next Lab

## Objetivo
Aprender Git e GitHub de forma prática e colaborativa, enquanto contribuímos para o projeto do laboratório.

## Estrutura do treinamento
Cada semana terá:
1. **Exercícios Git/GitHub**: comandos, branches, commits, PRs, merge e resolução de conflitos.
2. **Contribuições para o laboratório**: arquivos, scripts, documentação ou dados úteis para o projeto.

## Como participar
1. Fork do repositório central.
2. Clonar para seu computador.
3. Criar branch com seu nome e semana: `semana-1/nome`
4. Resolver exercícios e adicionar contribuições na pasta `contribuicoes/`.
5. Commit com mensagens claras: `feat: descrição do que foi feito`
6. Push da branch para seu fork.
7. Abrir Pull Request (PR) para o repositório central.
8. Revisar PRs de colegas.
9. Merge aprovado pelo responsável.

## Boas práticas
- Mensagens de commit claras e descritivas.
- Manter branch main limpa.
- Organizar arquivos na pasta correta de cada semana.
- Usar `.gitignore` para arquivos desnecessários.
```
